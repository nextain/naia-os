# 이슈 기반 개발 사이클 (Issue-Driven Development)

> 미러: `.agents/workflows/issue-driven-development.yaml`

## 목적

업스트림 우선, 코드 중심의 피처 개발 방법론. 새 기능, 대규모 버그 수정 등 피처 수준 작업의 기본 워크플로우입니다.

단순 변경(오타, 설정값, 간단한 지시)은 `development-cycle.yaml`을 사용합니다.

---

## 14 단계

### 사용자 참여 (프론트로딩)

| # | 단계 | 게이트 | 설명 |
|---|------|--------|------|
| 1 | **이슈** | — | GitHub 이슈 생성 또는 수령 |
| 2 | **이해** | ✓ | 사용자와 이해 확인 |
| 3 | **범위** | ✓ | 조사 범위와 깊이 정의 |
| 4 | **조사** | — | 코드 중심 조사 (연속 2회 클린 패스까지 반복) |
| 4.5 | **연구 산출물** | — | 조사 결과를 구조화된 요약으로 정리 → 이슈 댓글에 게시 (경량 작업 시 생략) |
| 4.7 | **주석 사이클** | — | 연구 + 초안 계획을 사용자에게 제시, 최대 3회 반복 (경량 작업 시 생략) |
| 5 | **계획** | ✓ | 계획 초안 작성, 사용자 승인 |

### AI 자율 (품질 게이트 포함)

| # | 단계 | 설명 |
|---|------|------|
| 6 | **빌드** | 계획에 따라 구현, 한 단계씩 |
| 7 | **리뷰** | 단계별 + 전체 반복 리뷰 (연속 2회 클린 패스) |
| 8 | **E2E 테스트** | 실제 앱/서버 실행, 타겟 테스트 먼저 → 전체 테스트 |
| 9 | **테스트 후 리뷰** | 테스트 통과 후 재리뷰 (연속 2회 클린 패스) |
| 10 | **동기화** | 컨텍스트 업데이트 + 교훈 반영 → 사용자 확인 (게이트) |
| 11 | **동기화 검증** | 컨텍스트 정확성 검증 (연속 2회 클린 패스) |
| 12 | **보고** | 사용자에게 결과 요약 |
| 13 | **커밋** | 이슈 번호 참조하여 커밋, PR 생성 |
| 14 | **종료** | 교훈 재검토 + 단계별 완료 보고 이슈 댓글 → 사용자 확인 후 close (게이트) |

## 핵심 원칙

- **업스트림 분석 우선**: 설계 전에 실제 코드를 먼저 읽는다
- **최소 수정**: 업스트림 패턴을 따르고, 커스터마이징은 오버레이로
- **추측 금지**: 절대 가정하지 않는다 — 실제 구현을 읽는다
- **구조적 문제 우선**: 테스트 실패 시 도구를 탓하기 전에 문서-코드 불일치를 먼저 확인
- **작동 코드 보존**: "개선"을 위해 작동하는 코드를 절대 깨뜨리지 않는다

## 연구 단계 (4.5 / 4.7)

### 연구 산출물 (Phase 4.5)

**생략 조건**: 경량 작업(비기능 변경) 또는 L1 범위에서 모듈 간 데이터 흐름이 없는 경우.

**절차**:
1. 조사 결과를 종합: 대상 코드 영역, 모듈 간 데이터 흐름, 발견한 참조 구현
2. 모든 주장에 file:line 인용 포함
3. 이슈 댓글에 `## Research Findings` 제목으로 게시 (영어)

### 주석 사이클 (Phase 4.7)

**최대 반복**: 3회

**절차**:
1. 연구 결과와 계획 초안을 사용자에게 제시
2. 사용자가 인라인 수정 또는 주석 추가
3. 수정 사항이 있으면: 계획만 수정 — 구현 코드 작성 금지
4. "주석 사이클 N/3" 추적
5. 사용자 승인 또는 최대 반복까지 반복

## 반복 리뷰

모든 리뷰 루프는 **연속 2회 클린 패스** 후 종료됩니다 (1회가 아님). 단일 클린 패스는 위음성일 수 있습니다.

**적용 시점 (5개)**:
1. **계획** 후 — 빌드 전 계획 리뷰
2. 각 **빌드** 단계 후 — 단계별 코드 리뷰 + 테스트
3. 전체 **빌드** 후 — 전체 변경사항 코드 리뷰
4. **E2E 테스트** 후 — 테스트 후 전체 코드 리뷰
5. **동기화** 후 — 컨텍스트 미러 정확성 검증

## 진행 파일 (세션 핸드오프)

진행 파일(`.agents/progress/*.json`)은 컨텍스트 압축과 세션 경계를 넘겨 다음 AI 세션이 상태를 잃지 않고 작업을 재개할 수 있게 합니다.

**Gitignored** — 세션 로컬 전용, 커밋되지 않음.

```json
{
  "issue": "#42",
  "title": "기능 설명",
  "project": "naia-os",
  "current_phase": "build",
  "gate_approvals": { "understand": "ISO-timestamp", ... },
  "decisions": [{ "decision": "...", "rationale": "...", "date": "..." }],
  "surprises": [],
  "blockers": [],
  "updated_at": "ISO-timestamp"
}
```
