# 다국어 지원 (i18n)

Naia Shell의 UI 번역과 AI 응답 언어 처리 방식.

## 지원 언어 (14개)

ko, en, ja, zh, fr, de, ru, es, ar, hi, bn, pt, id, vi

기본 fallback: **영어** (브라우저 언어 감지).

## i18n의 두 가지 레이어

### 1. UI 번역 (`i18n.ts`)

- 인라인 번역 맵: `Record<TranslationKey, Record<Locale, string>>`
- 감지 순서: 저장된 config > 브라우저 언어 > `"en"`
- 사용법: `t("settings.title")` → 현재 locale에 맞는 문자열 반환

### 2. AI 응답 언어 (`persona.ts`)

- `buildSystemPrompt`가 `"IMPORTANT: Respond in {언어명}"` 을 `context.locale` 기반으로 주입
- 기본 페르소나는 언어 중립: *"Speaks naturally in the user's preferred language"*
- locale fallback: `getLocale()` (브라우저 감지, 기본 `"en"`)

## Locale 의존 기능

| 기능 | 설명 | 한국어 전용 |
|------|------|:-----------:|
| 말투 (Speech Style) | 반말/존댓말 토글 | O |
| 호칭 (Honorific) | 한국식 접미사 (오빠, 님 등) | O |
| 감정 예시 (Emotion Examples) | 시스템 프롬프트의 인사 예시 | X (14개 언어 모두) |

말투와 호칭은 locale이 한국어가 아닐 때 설정 UI에서 숨겨짐.

## Locale 전파

locale이 변경되면 다음에 전달해야 함:

1. `buildSystemPrompt` (페르소나 컨텍스트)
2. `syncToOpenClaw` (게이트웨이 설정)
3. 전파 경로: ChatPanel, SettingsTab, OnboardingWizard, channel-sync

## 주요 파일

| 파일 | 역할 |
|------|------|
| `shell/src/lib/i18n.ts` | UI 번역 문자열 + locale 감지 |
| `shell/src/lib/persona.ts` | AI 프롬프트 언어 주입 |
| `shell/src/components/ChatPanel.tsx` | buildMemoryContext에서 locale 로딩 |
| `shell/src/components/SettingsTab.tsx` | locale UI + 저장 시 전파 |
| `shell/src/components/OnboardingWizard.tsx` | 온보딩 시 locale 전파 |
