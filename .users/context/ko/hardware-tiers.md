# Naia OS 하드웨어 요구사항

## 원칙

Naia OS는 로컬 우선 AI. 사용자 데이터는 절대 디바이스를 벗어나지 않는다.
사용자 텍스트를 보는 모든 AI 작업(대화, 기억, 임베딩, 팩트 추출, 모순 감지)은 **반드시 로컬 실행**.
GPU는 선택이 아닌 **제품 요구사항**.

## 하드웨어 티어

### 풀스펙 (128GB)

- **기준 HW**: Radeon AI MAX+ 395 (128GB 통합 메모리)
- omni 모델 (MiniCPM-o, 음성+비전+텍스트)
- 대형 대화 모델 (70B Q4, ~40GB)
- 기억 의식적 사고 (32B, ~20GB)
- 기억 자율신경계 (임베딩 + 4B, ~3.5GB)
- **전부 동시 상주, 클라우드 호출 0**
- 예상 합계: ~103GB, 여유 ~25GB

### 기본 (48GB)

- **기준 HW**: RTX 4090×2 (24GB×2) 또는 A40 (48GB) 또는 동급
- omni 모델 + 기억 자율신경계 + 기억 의식적 사고
- 대부분의 기능이 로컬에서 동작
- 대형 모델(70B) 동시 상주는 불가

### 마지노선 — omni 사용 시 (32GB)

- **기준 HW**: RTX 5090 (32GB) 또는 동급
- omni 모델 (최적화/양자화 필수) + 기억 자율신경계
- 기억 의식적 사고는 omni idle 시에만 또는 모델 교대 로드
- omni 최적화 성공 여부에 의존

### 텍스트 전용 — omni 없이 (16GB~24GB)

- **기준 HW**: RTX 4060 Ti 16GB / RTX 4070 Ti Super 16GB / RTX 3090 24GB
- 텍스트 전용 대화 모델 (8B Q4, ~6GB)
- 기억 자율신경계 (임베딩 0.5GB + 4B 3GB)
- STT/TTS 별도 경량 모델 (Whisper, Piper)
- omni 불가, 기억 의식적 사고는 모델 교대 로드
- 16GB: KV cache 포함 ~12.5GB, 여유 ~3.5GB (빡빡)
- 24GB: 여유 ~11GB (편안)
- **16GB는 최소 동작, 24GB 권장**

### 지원 대상 아님

- 16GB 미만 또는 GPU 없음
- 클라우드 AI 서비스 사용 권장

## 기억 시스템 GPU 배치

### 자율신경계 (상시, 매 턴)
- 임베딩 생성 (~0.5GB)
- 노이즈 필터링 + 팩트 추출 (4B 모델, ~3GB)
- 합계: **~3.5GB**

### 의식적 사고 (필요시, 비동기)
- 모순 감지, 기억 압축, 인출 판단
- 8B~32B 모델, 필요시 로드

### CPU 전용 (LLM 불필요, 상시)
- 에빙하우스 감쇠 계산 (decay.ts)
- 중요도 점수 (importance.ts)
- 지식그래프 가중치 관리 (knowledge-graph.ts)
- 통합 타이머/스케줄링

## STT/TTS 파이프라인 (omni 없을 때)

- STT: Whisper (로컬, ~1.5GB) 또는 faster-whisper
- TTS: Piper (CPU, 경량) 또는 edge-tts

## vllm 연동

vllm으로 여러 모델을 서빙, Gateway(any-llm)가 작업별 라우팅:
- `/chat` → 대화 모델
- `/memory/extract` → 4B 팩트 추출
- `/memory/contradiction` → 대형 모델
- `/memory/embed` → 임베딩 모델

## 경쟁사 대비 차별화

- AIRI: 단일 LLM, 작업별 모델 분리 없음
- mem0: 단일 LLM 설정
- **Naia: Gateway 멀티 프로바이더 라우팅 + 자율신경계/의식적 사고 분리**
