# Internacionalização (i18n)

Como a Naia Shell gere a interface multilingue e a língua de resposta da IA.

## Locales Suportadas (14)

ko, en, ja, zh, fr, de, ru, es, ar, hi, bn, pt, id, vi

Fallback padrão: **Inglês** (detetado a partir da língua do browser).

## Duas Camadas de i18n

### 1. Tradução da UI (`i18n.ts`)

- Mapa de tradução inline: `Record<TranslationKey, Record<Locale, string>>`  
- Ordem de deteção: configuração guardada > língua do browser > `"en"`  
- Utilização: `t("settings.title")` devolve a string localizada

### 2. Linguagem de Resposta da IA (`persona.ts`)

- `buildSystemPrompt` injeta `"IMPORTANT: Respond in {Language}"` com base em `context.locale`  
- Persona padrão é neutra em termos de língua: *"Fala naturalmente na língua preferida do utilizador"*  
- Fallback de locale: `getLocale()` (detecção do browser, padrão `"en"`)

## Funcionalidades Sensíveis à Língua

| Funcionalidade | Descrição | Apenas Coreano |
|----------------|-----------|:-------------:|
| Estilo de Fala | Alternância 반말/존댓말 | Sim |
| Honoríficos | Sufixos coreanos (오빠, 님, etc) | Sim |
| Exemplos de Emoção | Exemplo de saudação no system prompt | Não (todas as 14 locales) |

Estilo de fala e honoríficos ficam ocultos na UI de Definições quando a língua não é coreana.

## Propagação do Locale

Quando o locale muda, deve ser passado para:

1. `buildSystemPrompt` (contexto da persona)  
2. `syncToOpenClaw` (configuração do gateway)  
3. Ambos ocorrem em: ChatPanel, SettingsTab, OnboardingWizard, channel-sync

## Ficheiros Chave

| Ficheiro | Função |
|----------|-------|
| `shell/src/lib/i18n.ts` | Strings de tradução da UI + deteção de locale |
| `shell/src/lib/persona.ts` | Injeta língua no prompt da IA |
| `shell/src/components/ChatPanel.tsx` | `buildMemoryContext` com locale |
| `shell/src/components/SettingsTab.tsx` | UI de locale + propagação ao guardar |
| `shell/src/components/OnboardingWizard.tsx` | Propagação de locale durante setup |
