# Internationalization (i18n)

How Naia Shell handles multilingual UI and AI response language.

## Supported Locales (14)

ko, en, ja, zh, fr, de, ru, es, ar, hi, bn, pt, id, vi

Default fallback: **English** (detected from browser language).

## Two Layers of i18n

### 1. UI Translation (`i18n.ts`)

- Inline translation map: `Record<TranslationKey, Record<Locale, string>>`
- Detection order: saved config > browser language > `"en"`
- Usage: `t("settings.title")` returns localized string

### 2. AI Response Language (`persona.ts`)

- `buildSystemPrompt` injects `"IMPORTANT: Respond in {Language}"` based on `context.locale`
- Default persona is language-neutral: *"Speaks naturally in the user's preferred language"*
- Locale fallback: `getLocale()` (browser detection, default `"en"`)

## Locale-Aware Features

| Feature | Description | Scope |
|---------|------------|:-----:|
| Speech Style | casual/formal toggle | 11 locales with formal/informal distinction (ko, ja, de, fr, es, hi, vi, ru, pt, id, ar) |
| Honorific | Title/suffix (오빠, 先輩, Herr, etc) | Same 11 locales |
| Emotion Examples | Greeting example in system prompt | All 14 locales |

Speech style and honorific are hidden in Settings/Onboarding when the locale has no formal/informal distinction (en, zh, bn). Stored values are `"casual"` / `"formal"` (legacy `"반말"`/`"존댓말"` auto-migrated on startup).

## Locale Propagation

When locale changes, it must be passed to:

1. `buildSystemPrompt` (persona context)
2. `syncToGateway` (gateway config)
3. Both happen in: ChatPanel, SettingsTab, OnboardingWizard, channel-sync

## Key Files

| File | Role |
|------|------|
| `shell/src/lib/i18n.ts` | UI translation strings + locale detection |
| `shell/src/lib/persona.ts` | AI prompt language injection |
| `shell/src/components/ChatPanel.tsx` | buildMemoryContext with locale |
| `shell/src/components/SettingsTab.tsx` | Locale UI + save propagation |
| `shell/src/components/OnboardingWizard.tsx` | Locale propagation on setup |

  - Settings: normalized value storage + legacy migration
  - Onboarding: speechStyle step skip for non-formality locales (3)
