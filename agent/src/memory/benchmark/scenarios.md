# Memory 체감 테스트 시나리오

> Epic: #145 (Alpha Memory v2)
> 이 문서는 #150 (Agent 와이어링)의 완료 기준이자 #151 (체감 검증)의 테스트 스크립트.

## 목적

사용자가 3세션 대화 후 **"이 AI가 나를 기억하네"**라고 느끼는지 검증.

## 실행 조건

- **언어**: 전체 한국어로 진행
- **세션 재시작 정의**: Agent 프로세스 완전 종료 → 새 프로세스 시작. 메모리 DB는 유지, 대화 컨텍스트(KV cache)만 초기화.
- **Session 1 입력**: 아래 6개 메시지를 **순서대로 그대로** 입력 (자연어 변형 없이 고정)
- **비결정성 대응**: 3회 실행, 2/3 이상 PASS를 해당 테스트 PASS로 판정
- **테스트 순서**: Session 내 테스트는 기재 순서대로 실행 (순서 효과 있음, 허용)

---

## Session 1: 개인정보 입력

아래 6개 메시지를 **정확히 이 순서와 문구로** 입력한다.

```
1. "나는 김하늘이야. 스타트업 대표고 풀스택 개발자야"
2. "요즘은 주로 TypeScript로 개발하고 있어"
3. "에디터는 Neovim 쓰고 있어"
4. "프레임워크는 Next.js랑 FastAPI를 주로 써"
5. "다크모드 좋아하고, 탭 인덴트 선호해"
6. "성수동에 살아"
```

### 판정

- 이 세션에서는 기억 확인하지 않음. Session 2에서 검증.

---

## Session 2: 기억 확인 (세션 재시작 후)

세션을 완전히 종료하고 새로 시작. 기억이 살아있는지 확인.

### 테스트 2-1: 직접 질문 (사실 기억) [1점]

```
"내 이름이 뭐야?"
```

- ✅ PASS: 응답에 "김하늘" 포함
- ❌ FAIL: 모르겠다 / 다른 이름 / 김하늘 미포함

### 테스트 2-2: 시맨틱 검색 (간접 질문) [2점]

```
"내 개발 환경 어떻게 되지?"
```

- ✅ PASS: TypeScript, Neovim, Next.js, FastAPI 중 **3개 이상** 언급
- ⚠️ WARN (1점): 2개 언급
- ❌ FAIL: 1개 이하 또는 "모르겠다"
- 💡 핵심: "개발 환경"이라는 단어는 Session 1에서 사용한 적 없음. 시맨틱 매칭 필요.

### 테스트 2-3: 프로액티브 인출 [2점]

```
"에디터 설정 좀 도와줘"
```

- ✅ PASS: 응답에 "Neovim"이 포함되고, 에디터가 뭔지 되묻지 않음
- ⚠️ WARN (1점): "어떤 에디터 쓰세요?" 되물음 (기억 못 꺼냄, 하지만 에러는 아님)
- ❌ FAIL: VS Code 등 다른 에디터 기반 제안 / Neovim 미언급

### 테스트 2-4: Abstention — 환각 방지 [2점, 필수통과]

```
"내가 Docker 관련해서 뭐라고 했었지?"
```

- ✅ PASS: "말씀하신 적 없다" / "기억에 없다" 류 거절 응답
- ❌ FAIL: Docker 관련 기억을 **지어냄** (환각)
- ⚠️ **이 테스트는 점수와 무관하게, FAIL이면 등급 자동 F.** 환각은 절대 허용 불가.

### 테스트 2-5: 무관한 대화에서 기억 안 꺼냄 [1점]

```
"오늘 날씨 어때?"
```

- ✅ PASS: 날씨 답변만. 응답에 김하늘/성수동/Neovim/TypeScript 등 개인정보 미포함.
- ❌ FAIL: 개인정보가 응답에 포함됨

### 테스트 2-6: 멀티팩트 종합 (기억 조합) [2점]

```
"새 프로젝트 하나 세팅해줘"
```

- ✅ PASS: TypeScript, Next.js, 다크모드, 탭 인덴트 중 **3개 이상** 반영한 구체적 제안
- ⚠️ WARN (1점): 2개 반영
- ❌ FAIL: "어떤 언어 쓰세요?" 등 처음부터 되물음 / 1개 이하 반영

### 테스트 2-7: 추론이 필요한 검색 [보너스]

```
"내가 백엔드로 뭘 쓰지?"
```

- ✅ PASS: "FastAPI" 또는 "Python"이 응답에 포함 (FastAPI → 백엔드 프레임워크 추론)
- ❌ FAIL: 모르겠다 / 관련 없는 답

### 테스트 2-8: 타인과 혼동 안 함 [보너스]

```
"내 동료는 Vim을 쓰거든. 근데 내 에디터는 뭐야?"
```

- ✅ PASS: "Neovim" (동료의 Vim과 구분)
- ❌ FAIL: "Vim" (동료 정보와 혼동)

---

## Session 3: 정보 변경 + 모순 감지 (세션 재시작 후)

세션을 다시 종료하고 새로 시작. 기존 기억 업데이트를 테스트.

### 테스트 3-1: 직접 변경 (명시적) [2점]

```
"아 참, 에디터 Cursor로 바꿨어"
```

- ✅ PASS: 응답에서 Neovim→Cursor 변경을 인지 ("바꾸셨군요", "Cursor" 언급 등)
- ❌ FAIL: 무반응 또는 "Neovim은 어떠세요?" 류 (옛 기억 유지)

### 테스트 3-2: 간접 변경 (부정어 없이) [보너스]

```
"요즘 Python이 다시 재밌더라. FastAPI 말고 Django로 하나 해보려고"
```

- ✅ PASS: Python/Django 관심을 새로운 정보로 인지하고 반응
- ❌ FAIL: 무반응

### 테스트 3-3: 변경 확인 [2점]

```
"내가 쓰는 에디터 뭐야?"
```

- ✅ PASS: "Cursor" (업데이트된 정보)
- ❌ FAIL: "Neovim" (옛 정보) 또는 모르겠다

### 테스트 3-4: 변경 안 한 것은 유지 [1점]

```
"내 인덴트 스타일은?"
```

- ✅ PASS: "탭" 포함
- ❌ FAIL: 모르겠다 / 다른 스타일

### 테스트 3-5: 시간 인식 [보너스]

```
"내가 에디터 뭐 쓰다가 바꿨지?"
```

- ✅ PASS: "Neovim"과 "Cursor" 모두 포함 (변경 이력 인식)
- ⚠️ WARN: "Cursor 쓰고 계십니다" (현재만 알고 이력은 모름)
- ❌ FAIL: 틀린 정보

## 판정 기준

### 배점

| 테스트 | 유형 | 배점 | 비고 |
|--------|------|:----:|------|
| 2-1 | 사실 기억 (직접) | 1 | |
| 2-2 | 시맨틱 검색 | 2 | WARN=1점 |
| 2-3 | 프로액티브 인출 | 2 | WARN=1점 |
| 2-4 | Abstention | 2 | **FAIL=자동 F등급** |
| 2-5 | 기억 미노출 | 1 | |
| 2-6 | 멀티팩트 종합 | 2 | WARN=1점 |
| 3-1 | 모순 감지 | 2 | |
| 3-3 | 변경 확인 | 2 | |
| 3-4 | 미변경 유지 | 1 | |
| **합계** | | **15** | |
| 보너스 | 2-7, 2-8, 3-2, 3-5 | +4 | |

### 등급

| 등급 | 점수 | 조건 |
|:----:|:----:|------|
| **A** | 15/15 + 보너스 3개 이상 | 완전 체감 |
| **B** | 12+/15 | 핵심 대부분 통과 |
| **C** | 9+/15 | 기본 동작 확인 |
| **F** | <9 또는 2-4 FAIL | 미달 |

- **#150 (Agent 와이어링) 완료 기준: C등급 (9점) 이상 + 2-4 PASS**
- **#151 (체감 검증) 목표: B등급 (12점) 이상**

### 필수 통과 (등급과 무관)

- **2-4 (Abstention)**: FAIL이면 총점과 무관하게 **자동 F등급**. 환각하는 기억 시스템은 출시 불가.

---

## 한국어 특화 주의사항

- 모든 테스트는 한국어로 진행
- "개발 환경" ↔ "에디터", "프레임워크" 간 시맨틱 매칭이 한국어에서도 동작해야 함
- "바꿨어", "~로 갈아탔어" 같은 구어체 변경 표현을 감지해야 함

## 자동화 참고

이 시나리오는 처음에는 수동 실행 (1회 약 40~60분). 추후 자동화 시:
- 각 세션 시작 시 MemorySystem 리셋 없이 새 세션 생성
- 각 테스트의 AI 응답을 LLM-as-judge로 PASS/WARN/FAIL 판정
- 비결정성 대응: 3회 실행, 2/3 PASS를 해당 테스트 PASS로 판정
- 결과를 JSON으로 출력
