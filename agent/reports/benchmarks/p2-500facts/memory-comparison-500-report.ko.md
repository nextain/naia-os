# 메모리 시스템 500개 Fact 스케일 벤치마크 보고서

> 날짜: 2026-04-01
> 이슈: #189
> 벤치마크: 500개 fact, 106개 쿼리, 키워드 심판
> 테스트 LLM: Gemini 2.5 Flash, Qwen3-8B (ollama 로컬)
> 시스템: Naia, mem0, Naia, SAP

## 요약

Naia 메모리 시스템이 500개 fact 환경에서 Gemini/Qwen3 양쪽 LLM으로 **핵심 정확도 70-72%**를 달성했습니다. 모든 시스템이 100-fact 대비 점수 하락을 보였으며, 500개 fact(2% 검색비율)가 실질적으로 더 어려운 시험임을 확인했습니다.

## 핵심 발견

### 1. Naia vs mem0 스케일 역전
100 facts에서 Naia(88%) > mem0(84%)였지만, 500 facts에서 mem0(74%) > Naia(70-72%)로 역전. Naia의 뇌과학 레이어(감쇠, 중요도 게이팅)가 대규모에서 과다 필터링하는 것으로 추정. **감쇠 파라미터의 스케일 적응이 1순위 최적화 대상.**

### 2. Naia 시간 추론 강점
Naia가 temporal_history에서 4/4 (보너스)로 유일하게 만점. 전용 시간 처리 메커니즘이 있는 것으로 보임.

### 3. 일관된 강점
- **무관 정보 격리**: 모든 시스템 100%, 모든 스케일 — 해결된 문제
- **Naia 능동적 기억 적용**: 60-70%로 지속적 1위 — KG spreading activation 효과

### 4. 공통 약점
- **환각 방지**: 모든 시스템 67-75% — 기억이 많을수록 거짓 관련성 증가

## 권장사항

1. **감쇠 스케일 튜닝**: BASE_DECAY 줄이거나 스케일 인지 감쇠 추가 (현재 0.08이 500+ fact에서 과도할 수 있음)
2. **하이브리드 검색**: 키워드 사전 필터 + 벡터 재순위로 스케일 정밀도 확보
3. **시간 인덱스**: 전용 시간 기반 검색 (Naia 접근법 참고)
4. **재순위 레이어**: 검색 후 LLM 주입 전 관련성 재점수
5. **목표**: 500 facts에서 80%+ 달성 후 대규모 프로덕션 준비 판정
