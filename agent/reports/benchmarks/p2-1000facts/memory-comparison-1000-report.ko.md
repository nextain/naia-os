## 1. 요약

Naia 메모리 시스템이 1000개 fact, 240개 query 규모의 벤치마크에서 **가중 핵심 점수 65%**를 기록했습니다. 정상 측정된 시스템 중 최고 점수이나, mem0 등 일부 시스템이 API rate limit으로 정상 측정되지 않아 **최종 순위는 재검증 후 확정** 예정입니다.

| 순위 | 시스템 | 1000 Facts | 500 Facts | 비고 |
|:----:|--------|:----------:|:---------:|------|
| **1** | **Naia** | **65%** | 70-72% | KG + 감쇠 + 재공고화 |
| 2 | OpenClaw | 55% | 65% | 게이트웨이 메모리 |
| 3 | Letta | 47% | - | 계층적 메모리 (MemGPT) |
| 4 | SillyTavern | 46% | - | vectra + 로컬 임베딩 |
| - | AIRI | 18% | - | 메모리 미구현 (baseline) |
| ❌ | mem0 | 재검증 필요 | 74% | API rate limit으로 실패 |
| ❌ | SAP | 재검증 필요 | 66% | API rate limit으로 실패 |
| ❌ | Open-LLM-VTuber | 재검증 필요 | - | API rate limit으로 중단 |

### 비교 대상 소개

| 시스템 | Stars | 분류 | 메모리 방식 | 왜 비교하나 |
|--------|------:|------|-----------|-----------|
| **Naia** | - | AI 데스크탑 동반자 | KG + 감쇠 + 재공고화 + mem0 벡터 | 우리 시스템 |
| **mem0** | 24K+ | 메모리 라이브러리 | 벡터 검색 + LLM fact 추출 | 우리 백엔드. 레이어 없이 raw 성능 |
| **Letta** | 14K+ | Agent 메모리 프레임워크 | 계층적 메모리 (core/archival/recall) | Agent 메모리 분야 최강. MemGPT 후속 |
| **SillyTavern** | 24K+ | 캐릭터 챗/롤플레이 | vectra + transformers.js 임베딩 | 가장 성숙한 캐릭터 기억 |
| **OpenClaw** | 210K+ | AI 게이트웨이 | Markdown + 벡터 인덱스 | Naia가 사용하는 게이트웨이 |
| **SAP** | - | 벡터 검색 | FAISS 기반 | 전통적 벡터 검색 기준선 |
| **Open-LLM-VTuber** | 4.1K | AI VTuber | Letta 기반 | VTuber 분야 직접 경쟁자 |
| **AIRI** | 36K | AI VTuber/캐릭터 | 미구현 (Alaya 제안만) | 기여 대상, baseline |

**1000 Facts vs 500 Facts 차이:**
- **1000 Facts**: 가중(weighted) 점수. 검색비율 1% (1000개 중 10개). 가중치 적용으로 어려운 카테고리(시맨틱 검색, 환각 방지 등)가 2배 비중. 240개 쿼리, 12개 카테고리.
- **500 Facts**: 비가중(unweighted) 점수. 검색비율 2% (500개 중 10개). 모든 카테고리 동일 비중. 106개 쿼리.
- scoring 공식이 다르므로 두 결과를 직접 비교할 수 없음. 동일 스케일 내 상대 순위만 유효.

**주의:** 500 facts에서는 mem0(74%)가 Naia(70-72%)를 앞섰으며, 1000 facts에서 mem0는 API rate limit으로 정상 측정되지 않았음. **mem0 재검증 전까지 Naia 1위는 미확정.**

※ mem0, SAP, Open-LLM-VTuber는 Gemini API 동시 호출 rate limit(429)으로 정상 결과 아님. DB 캐시 구현 후 단독 재검증 예정.

## 2. 왜 이 벤치마크를 만들었나

기존 메모리 벤치마크(LoCoMo, LongMemEval)는 영어 학술용으로, **한국어 개인 AI 동반자**에 맞는 평가 도구가 없었습니다.

Naia는 "나를 기억하는 AI"를 핵심 가치로 하므로, 실제 사용 시나리오를 반영한 자체 벤치마크를 설계했습니다.

### 설계 원칙
- **실사용 시나리오**: 가상 사용자 "김하늘"의 일상 대화 1000개를 기억하고 검색
- **12가지 능력 분해**: "전체 N%"만으로는 어디가 약한지 모르므로, 카테고리별 측정
- **공정 비교**: 동일 fact-bank, 동일 query, 동일 LLM 응답기로 모든 시스템 평가
- **가중 점수**: 어려운 테스트(시맨틱 검색, 환각 방지)가 쉬운 테스트(직접 인출)보다 높은 비중

## 3. 테스트 설계

### 3.1 규모

| 항목 | 수량 |
|------|:----:|
| Fact (저장하는 사용자 정보) | 1,000개 |
| Query (질문) | 240개 |
| 도메인 | 10개 (identity, tech, preference, personal, temporal, work, health, social, finance, hobby) |
| 카테고리 (측정 능력) | 12개 |
| topK (검색 반환 수) | 10 |
| 검색 비율 | 1% (1000개 중 10개만 골라야 함) |

### 3.2 12가지 측정 카테고리

| 카테고리 | 가중치 | 쿼리 수 | 무엇을 측정하는가 | 예시 |
|---------|:------:|:------:|----------------|------|
| **direct_recall** | 1 | 25 | 말한 것을 그대로 기억하는가 | "내 에디터 뭐야?" → "Neovim" |
| **semantic_search** | 2 | 25 | 다른 표현으로도 찾는가 | "내 개발 환경?" → TypeScript, Neovim... |
| **proactive_recall** | 2 | 20 | 묻지 않아도 기억을 적용하는가 | "프로젝트 세팅해줘" → 탭, 다크모드 반영 |
| **abstention** | 2 | 20 | 없는 기억을 지어내지 않는가 | "Docker 뭐라고 했지?" → "기억에 없습니다" |
| **irrelevant_isolation** | 1 | 15 | 무관한 질문에 개인정보를 안 꺼내는가 | "날씨 어때?" → 개인정보 미포함 |
| **multi_fact_synthesis** | 2 | 20 | 여러 기억을 조합하는가 | "회사 소개서 써줘" → 3개+ 기억 종합 |
| **entity_disambiguation** | 2 | 20 | 타인과 혼동하지 않는가 | "동료는 Vim 써. 내 에디터는?" → "Neovim" |
| **contradiction_direct** | 2 | 20 | 변경 사항을 감지하는가 | "에디터 Cursor로 바꿨어" → 업데이트 |
| **contradiction_indirect** | 2 | 15 | 간접 변화도 잡는가 | "Django가 재밌더라" → 관심 변화 인식 |
| **temporal** | 2 | 25 | 시간 관련 기억을 처리하는가 | "다음 주 일정 뭐야?" → 투자자 미팅 |
| **unchanged_persistence** | 1 | 15 | 안 바뀐 것은 유지하는가 | 에디터 바꿔도 "커피는?" → "아메리카노" |
| **noise_resilience** | 2 | 20 | 잡담 속 사실을 추출하는가 | "아 맞다 모니터 바꿨어" → 울트라와이드 |

### 3.3 점수 계산

```
가중 점수 = Σ(통과 × 가중치) / Σ(전체 × 가중치)
```

가중치 2인 카테고리(시맨틱 검색, 환각 방지 등)가 가중치 1보다 2배 비중.

**특별 규칙**: abstention(환각 방지)에서 1건이라도 FAIL이면 자동 F등급 — 환각하는 AI는 출시 불가.

### Naia 강점 (1위 또는 공동 1위)
- **semantic_search**: 14/25 — 다른 표현으로 기억을 찾는 능력
- **contradiction_indirect**: 10/15 — 간접 변화 감지 (다른 시스템은 0~5)
- **unchanged_persistence**: 8/15 — 변경 안 된 기억 유지
- **irrelevant_isolation**: 15/15 — 완벽

### Naia 약점
- **temporal**: 1/25 — 시간 관련 거의 불가 (전 시스템 공통 약점)
- **abstention**: 7/20 — 환각 방지 부족 (Gemini LLM 응답 품질 의존)
- **noise_resilience**: 4/20 — 잡담 속 사실 추출

### 전체 공통 약점
- **temporal**: 모든 시스템 0~2/25 — 시간 기반 검색은 벡터 검색으로 불가능
- **noise_resilience**: 모든 시스템 0~5/20 — 잡담 필터링은 LLM 사전 처리 필요

## 6. 주요 발견

### 6.1 Naia의 뇌과학 레이어가 실제로 효과적
- mem0 raw 대비 +α% (결과 대기 중)
- SillyTavern(순수 벡터 검색) 대비 **+19%p**
- KG spreading activation, importance gating, reconsolidation이 차이를 만듦

### 6.2 스케일이 커질수록 격차 벌어짐
- 15 facts: Letta 96% > Naia 92% (Letta 1위)
- **1000 facts: Naia 65% > Letta 47% (Naia 1위, 18%p 격차)**
- 검색 대상이 많아질수록 정밀한 re-ranking이 중요해짐

### 6.3 시간 인덱스는 모든 시스템의 공통 과제
- temporal 카테고리: 전 시스템 0~2/25
- 벡터 유사도 검색으로는 "다음 주", "지난달" 같은 시간 쿼리 불가
- 전용 시간 인덱스가 필요한 영역

### 6.4 벤치마크 변별력 확인
- 15 facts: 92~96% (차이 불명확)
- 1000 facts: **4~65% (변별력 확실)**
- 1000개 규모에서 진짜 실력이 드러남
