## Naia কী?

Naia একটি ব্যক্তিগত AI OS যা প্রতিটি ব্যক্তিকে তাদের AI-এর উপর সম্পূর্ণ সার্বভৌমত্ব দেয়। কোন AI ব্যবহার করবেন তা বেছে নিন (লোকাল মডেলসহ), এর মেমরি ও ব্যক্তিত্ব স্থানীয়ভাবে কনফিগার করুন, এর 3D অবতার ও কণ্ঠস্বর কাস্টমাইজ করুন — সবকিছু আপনার মেশিনে থাকে, আপনার নিয়ন্ত্রণে।

এটি শুধু আরেকটি AI টুল নয়। এটি একটি অপারেটিং সিস্টেম যেখানে আপনার AI বাস করে, বেড়ে ওঠে, এবং আপনার পাশে কাজ করে। আজ এটি 3D অবতারসহ একটি ডেস্কটপ OS। আগামীকাল — রিয়েল-টাইম ভিডিও অবতার, গান গাওয়া, গেমিং, এবং শেষ পর্যন্ত আপনার নিজের Physical AI (অ্যান্ড্রয়েড OS)।

### মূল দর্শন

- **AI সার্বভৌমত্ব** — আপনি আপনার AI বেছে নেন। ক্লাউড বা লোকাল। OS নির্দেশ দেয় না।
- **সম্পূর্ণ নিয়ন্ত্রণ** — মেমরি, ব্যক্তিত্ব, সেটিংস — সব স্থানীয়ভাবে সংরক্ষিত। কোনো ক্লাউড নির্ভরতা নেই।
- **আপনার নিজের AI** — অবতার, কণ্ঠস্বর, নাম, ব্যক্তিত্ব কাস্টমাইজ করুন। এটিকে সত্যিই আপনার করে তুলুন।
- **সর্বদা জীবিত** — AI ব্যাকগ্রাউন্ডে ২৪/৭ চলে, বার্তা গ্রহণ করে এবং কাজ করে এমনকি আপনি দূরে থাকলেও।
- **ভবিষ্যৎ দৃষ্টি** — VRM 3D অবতার → রিয়েল-টাইম ভিডিও অবতার → একসাথে গান গাওয়া ও গেমিং → Physical AI

### বৈশিষ্ট্যসমূহ

- **3D অবতার** — আবেগ প্রকাশ (আনন্দ/দুঃখ/বিস্ময়/চিন্তা) এবং লিপ-সিঙ্কসহ VRM ক্যারেক্টার
- **AI স্বাধীনতা** — ৭টি ক্লাউড প্রদানকারী (Gemini, Claude, GPT, Grok, zAI) + লোকাল AI (Ollama) + Claude Code CLI
- **লোকাল-ফার্স্ট** — মেমরি, ব্যক্তিত্ব, সব সেটিংস আপনার মেশিনে সংরক্ষিত
- **টুল এক্সিকিউশন** — ৮টি টুল: ফাইল পড়া/লেখা, টার্মিনাল, ওয়েব সার্চ, ব্রাউজার, সাব-এজেন্ট
- **৭০+ দক্ষতা** — ৭টি বিল্ট-ইন + ৬৩টি কাস্টম + ৫,৭০০+ ClawHub কমিউনিটি দক্ষতা
- **কণ্ঠস্বর** — ৫টি TTS প্রদানকারী + STT + লিপ-সিঙ্ক। আপনার AI-কে আপনার পছন্দের কণ্ঠস্বর দিন।
- **১৪টি ভাষা** — কোরিয়ান, ইংরেজি, জাপানি, চীনা, ফরাসি, জার্মান, রাশিয়ান এবং আরও
- **সর্বদা চালু** — OpenClaw গেটওয়ে ডেমন আপনার AI-কে ব্যাকগ্রাউন্ডে চালু রাখে
- **চ্যানেল ইন্টিগ্রেশন** — Discord DM-এর মাধ্যমে যেকোনো সময়, যেকোনো জায়গায় AI-এর সাথে কথা বলুন
- **৪-স্তরের নিরাপত্তা** — T0 (পড়া) থেকে T3 (বিপজ্জনক), প্রতি-টুল অনুমোদন, অডিট লগ
- **ব্যক্তিগতকরণ** — নাম, ব্যক্তিত্ব, কথা বলার ধরন, অবতার, থিম (৮ ধরনের)

## কেন Naia?

অন্যান্য AI টুলগুলো শুধু "টুল"। Naia হলো **"আপনার নিজের AI"**।

| | অন্যান্য AI টুল | Naia |
|---|----------------|------|
| **দর্শন** | AI-কে টুল হিসেবে ব্যবহার করা | AI-কে OS দিয়ে দাও। একসাথে থাকো। |
| **লক্ষ্য** | শুধু ডেভেলপার | সবাই যারা নিজের AI চায় |
| **AI নির্বাচন** | প্ল্যাটফর্ম ঠিক করে | ৭ ক্লাউড + লোকাল AI — আপনি ঠিক করুন |
| **ডেটা** | ক্লাউডে আটকে | মেমরি, ব্যক্তিত্ব, সেটিংস সব লোকাল |
| **অবতার** | নেই | VRM 3D ক্যারেক্টার + আবেগ + লিপ-সিঙ্ক |
| **কণ্ঠস্বর** | শুধু টেক্সট বা বেসিক TTS | ৫ TTS + STT + আপনার AI-এর নিজস্ব কণ্ঠস্বর |
| **ডিপ্লয়মেন্ট** | npm / brew / pip | ডেস্কটপ অ্যাপ বা বুটেবল USB OS |
| **প্ল্যাটফর্ম** | macOS / CLI / ওয়েব | Linux নেটিভ ডেস্কটপ → ভবিষ্যৎ: Physical AI |
| **খরচ** | আলাদা API কী প্রয়োজন | শুরু করতে বিনামূল্যে ক্রেডিট, লোকাল AI সম্পূর্ণ বিনামূল্যে |

## OpenClaw-এর সাথে সম্পর্ক

Naia [OpenClaw](https://github.com/openclaw-ai/openclaw) ইকোসিস্টেমের উপর নির্মিত, কিন্তু এটি মৌলিকভাবে একটি ভিন্ন পণ্য।

| | OpenClaw | Naia |
|---|---------|---------|
| **রূপ** | CLI ডেমন + টার্মিনাল | ডেস্কটপ অ্যাপ + 3D অবতার |
| **লক্ষ্য** | ডেভেলপার | সবাই |
| **UI** | নেই (টার্মিনাল) | Tauri 2 নেটিভ অ্যাপ (React + Three.js) |
| **অবতার** | নেই | VRM 3D ক্যারেক্টার (আবেগ, লিপ-সিঙ্ক, দৃষ্টি) |
| **LLM** | একক প্রদানকারী | মাল্টি-প্রদানকারী ৭ + রিয়েল-টাইম সুইচিং |
| **ভয়েস** | TTS ৩ (Edge, OpenAI, ElevenLabs) | TTS ৫ (+Google, Nextain) + STT + অবতার লিপ-সিঙ্ক |
| **আবেগ** | নেই | ৬টি আবেগ মুখের অভিব্যক্তিতে ম্যাপ করা |
| **অনবোর্ডিং** | CUI | GUI + VRM অবতার নির্বাচন |
| **খরচ ট্র্যাকিং** | নেই | রিয়েল-টাইম ক্রেডিট ড্যাশবোর্ড |
| **বিতরণ** | npm install | Flatpak / AppImage / DEB / RPM + OS ইমেজ |
| **বহুভাষিক** | ইংরেজি CLI | ১৪-ভাষা GUI |
| **চ্যানেল** | সার্ভার বট (মাল্টি-চ্যানেল) | Naia-নিবেদিত Discord DM বট |

**OpenClaw থেকে যা নেওয়া হয়েছে:** ডেমন আর্কিটেকচার, টুল এক্সিকিউশন ইঞ্জিন, চ্যানেল সিস্টেম, দক্ষতা ইকোসিস্টেম (৫,৭০০+ Clawhub দক্ষতা সামঞ্জস্যপূর্ণ)

**Naia যা নতুন তৈরি করেছে:** Tauri Shell, VRM অবতার সিস্টেম, মাল্টি-LLM এজেন্ট, আবেগ ইঞ্জিন, TTS/STT ইন্টিগ্রেশন, অনবোর্ডিং উইজার্ড, খরচ ট্র্যাকিং, Nextain অ্যাকাউন্ট ইন্টিগ্রেশন, মেমরি সিস্টেম (STM/LTM), নিরাপত্তা স্তরসমূহ

## প্রসঙ্গ ডকুমেন্ট (Dual-directory Architecture)

AI এজেন্ট এবং মানব ডেভেলপারদের জন্য দ্বৈত ডকুমেন্টেশন কাঠামো। `.agents/`-এ AI-এর জন্য টোকেন-দক্ষ JSON/YAML আছে, `.users/`-এ মানুষের জন্য কোরিয়ান Markdown আছে।

| AI প্রসঙ্গ (`.agents/`) | মানব ডকুমেন্ট (`.users/`) | বিবরণ |
|---|---|---|
| [`context/agents-rules.json`](../.agents/context/agents-rules.json) | [`context/agents-rules.md`](../.users/context/agents-rules.md) | প্রকল্প নিয়ম (SoT) |

## দ্রুত শুরু

### পূর্বশর্ত

- Linux (Bazzite, Ubuntu, Fedora ইত্যাদি)
- Node.js 22+, pnpm 9+
- Rust stable (Tauri বিল্ডের জন্য)
- সিস্টেম প্যাকেজ: `webkit2gtk4.1-devel libappindicator-gtk3-devel librsvg2-devel` (Fedora)

### ডেভেলপমেন্ট চালানো

```bash
# নির্ভরতা ইনস্টল
cd shell && pnpm install
cd ../agent && pnpm install

# Tauri অ্যাপ চালান (Gateway + Agent অটো-স্পন)
cd ../shell && pnpm run tauri dev
```

অ্যাপ চালু হলে, স্বয়ংক্রিয়ভাবে:
1. OpenClaw Gateway হেলথ চেক → চলমান থাকলে পুনঃব্যবহার, না হলে অটো-স্পন
2. Agent Core স্পন (Node.js, stdio সংযোগ)
3. অ্যাপ বন্ধ হলে, শুধু অটো-স্পন করা Gateway বন্ধ হয়

### টেস্ট

```bash
cd shell && pnpm test                # Shell ইউনিট টেস্ট
cd agent && pnpm test                # Agent ইউনিট টেস্ট
cd agent && pnpm exec tsc --noEmit   # টাইপ চেক
cargo test --manifest-path shell/src-tauri/Cargo.toml  # Rust টেস্ট

# E2E (Gateway + API কী প্রয়োজন)
cd shell && pnpm run test:e2e:tauri
```

flatpak install --user flathub org.freedesktop.Platform//24.08 org.freedesktop.Sdk//24.08
flatpak-builder --user --install --force-clean build-dir flatpak/io.nextain.naia.yml

## নিরাপত্তা মডেল

Naia **গভীর প্রতিরক্ষা (Defense in Depth)** নিরাপত্তা মডেল প্রয়োগ করে:

| স্তর | সুরক্ষা |
|------|---------|
| OS | Bazzite অপরিবর্তনীয় rootfs + SELinux |
| Gateway | OpenClaw ডিভাইস প্রমাণীকরণ + টোকেন স্কোপ |
| Agent | ৪-স্তরের অনুমতি (T0~T3) + প্রতি-টুল ব্লকিং |

| অডিট | SQLite অডিট লগ (সমস্ত টুল এক্সিকিউশন রেকর্ড করা) |

## মেমরি সিস্টেম

- **স্বল্পমেয়াদী মেমরি (STM):** বর্তমান সেশন কথোপকথন (Zustand + SQLite)

## OS আপডেট

Naia OS [Bazzite](https://github.com/ublue-os/bazzite) (Fedora Atomic)-এর উপর নির্মিত। আপডেটগুলো **পারমাণবিক এবং নিরাপদ**:

- **স্বয়ংক্রিয়**: সাপ্তাহিক রিবিল্ড সর্বশেষ Bazzite নিরাপত্তা প্যাচ ও আপডেট নেয়
- **পারমাণবিক**: নতুন ইমেজ বর্তমানটির পাশে ডিপ্লয় হয় — ব্যর্থ হলে পুরনো ইমেজ অক্ষত থাকে
- **রোলব্যাক**: তাৎক্ষণিক পুনরুদ্ধারের জন্য GRUB মেনু থেকে আগের সংস্করণ নির্বাচন করুন
- **আমাদের ওভারলে**: শুধু প্যাকেজ (fcitx5, ফন্ট) + Naia Shell (Flatpak, স্যান্ডবক্স) + ব্র্যান্ডিং কনফিগ যোগ করে — কার্নেল, বুটলোডার বা systemd কোর কখনো স্পর্শ করে না

```
Bazzite বেস আপডেট → সাপ্তাহিক অটো-রিবিল্ড → কন্টেইনার স্মোক টেস্ট → ISO রিবিল্ড → R2 আপলোড
                                                                      ↘ GHCR পুশ → ব্যবহারকারী bootc আপডেট
```

## উন্নয়ন প্রক্রিয়া

### ফিচার উন্নয়ন (ডিফল্ট) — Issue-Driven Development

```
ISSUE → UNDERSTAND → SCOPE → INVESTIGATE → PLAN → BUILD → REVIEW → E2E → SYNC → COMMIT
```

- **৩টি বাধ্যতামূলক গেট** — UNDERSTAND, SCOPE এবং PLAN-এ ব্যবহারকারীর নিশ্চিতকরণ প্রয়োজন
- **Plan অনুমোদনের পর** — AI না থেমে BUILD থেকে COMMIT পর্যন্ত একটানা চালায়
- **নীতি** — আগে upstream কোড পড়ুন (অনুমান নিষিদ্ধ)। ন্যূনতম পরিবর্তন। কাজ করা কোড কখনো ভাঙবেন না।
- **কমিট** — ইংরেজি, `<type>(<scope>): <description>`
- **ফর্ম্যাটার** — Biome (ট্যাব, ডাবল কোট, সেমিকোলন)

## AI-নেটিভ ওপেন সোর্স

2025–2026 সালে বেশিরভাগ ওপেন সোর্স প্রকল্প AI অবদানের বিরুদ্ধে প্রতিরক্ষা করছে। **Naia বিপরীত পন্থা নেয়**: AI-সহায়িত অবদান যেন ডিফল্টভাবে উচ্চ মানের হয় সেভাবে প্রকল্প ডিজাইন করা।

> **"AI-এর বিরুদ্ধে প্রতিরক্ষা নয়, AI-এর সাথে ডিজাইন করো।"**

### কিভাবে কাজ করে

```
ব্যক্তি (যেকোনো ভাষা) → AI → Git (ইংরেজি) → AI → ব্যক্তি (যেকোনো ভাষা)
```

- **আপনার ভাষায় issue এবং PR লিখুন** — AI সবকিছু অনুবাদ করে
- **অবদানকারী এবং রক্ষণাবেক্ষণকারী উভয়ই AI ব্যবহার করেন** — কোডিং, রিভিউ, ট্রায়াজ
- **সমৃদ্ধ `.agents/` প্রসঙ্গ** AI-কে প্রকল্প গভীরভাবে বুঝতে সাহায্য করে — AI-এর ভালো বোঝাপড়া মানে উচ্চতর অবদান মান
- **10 ধরনের অবদান** — অনুবাদ, দক্ষতা, ফিচার, বাগ, কোড, ডকুমেন্টেশন, পরীক্ষা, ডিজাইন, নিরাপত্তা, প্রসঙ্গ
- **মাতৃভাষায় কাজের লগ** — আপনার ভাষায় একটি ব্যক্তিগত রেপো রাখুন; AI অনুবাদের মাধ্যমে Git ইতিহাস পর্যালোচনা করুন

এটি শুধু একটি নীতি নয়। এটি আর্কিটেকচার। `.agents/` ডিরেক্টরি, ট্রিপল-মিরর ডকুমেন্টেশন, এবং লাইসেন্স সুরক্ষা নিয়ম সবই AI সহযোগিতাকে কাঠামোগত করতে ডিজাইন করা হয়েছে, দুর্ঘটনাজনিত নয়।
