## ما هو Naia؟

Naia هو نظام تشغيل ذكاء اصطناعي شخصي يمنح الأفراد سيادة كاملة على ذكائهم الاصطناعي. اختر أي ذكاء اصطناعي تريد استخدامه (بما في ذلك النماذج المحلية)، وقم بتكوين ذاكرته وشخصيته محلياً، وخصص صورته الرمزية ثلاثية الأبعاد وصوته — كل شيء يبقى على جهازك، تحت سيطرتك.

هذه ليست مجرد أداة ذكاء اصطناعي أخرى. إنه نظام تشغيل يعيش فيه ذكاؤك الاصطناعي، وينمو، ويعمل بجانبك. اليوم هو نظام تشغيل سطح مكتب مع صورة رمزية ثلاثية الأبعاد. غداً — صور رمزية فيديو في الوقت الحقيقي، الغناء، الألعاب، وفي النهاية Physical AI (نظام تشغيل أندرويد) خاص بك.

### الفلسفة الأساسية

- **سيادة الذكاء الاصطناعي** — أنت تختار ذكاءك الاصطناعي. سحابي أو محلي. النظام لا يفرض.
- **تحكم كامل** — الذاكرة، الشخصية، الإعدادات — كلها مخزنة محلياً. بدون اعتماد على السحابة.
- **ذكاؤك الاصطناعي** — خصص الصورة الرمزية والصوت والاسم والشخصية. اجعله ملكك حقاً.
- **دائم الحياة** — يعمل الذكاء الاصطناعي على مدار الساعة في الخلفية، يستقبل الرسائل وينجز المهام حتى عندما تكون بعيداً.
- **رؤية المستقبل** — صور رمزية VRM ثلاثية الأبعاد → صور رمزية فيديو في الوقت الحقيقي → الغناء والألعاب معاً → Physical AI

### الميزات

- **صورة رمزية ثلاثية الأبعاد** — شخصية VRM مع تعبيرات عاطفية (فرح/حزن/دهشة/تفكير) ومزامنة الشفاه
- **حرية الذكاء الاصطناعي** — 7 مزودين سحابيين (Gemini، Claude، GPT، Grok، zAI) + ذكاء اصطناعي محلي (Ollama) + Claude Code CLI
- **محلي أولاً** — الذاكرة والشخصية وجميع الإعدادات مخزنة على جهازك
- **تنفيذ الأدوات** — 8 أدوات: قراءة/كتابة الملفات، الطرفية، البحث على الويب، المتصفح، الوكيل الفرعي
- **70+ مهارة** — 7 مدمجة + 63 مخصصة + 5,700+ مهارة مجتمع ClawHub
- **الصوت** — 5 مزودي TTS + STT + مزامنة الشفاه. امنح ذكاءك الاصطناعي الصوت الذي تريده.
- **14 لغة** — الكورية، الإنجليزية، اليابانية، الصينية، الفرنسية، الألمانية، الروسية والمزيد
- **دائم التشغيل** — خفي بوابة OpenClaw يبقي ذكاءك الاصطناعي يعمل في الخلفية
- **تكامل القنوات** — تحدث مع ذكائك الاصطناعي عبر Discord DM، في أي وقت ومن أي مكان
- **أمان من 4 مستويات** — T0 (قراءة) إلى T3 (خطير)، موافقة لكل أداة، سجلات تدقيق
- **التخصيص** — الاسم، الشخصية، أسلوب الكلام، الصورة الرمزية، السمة (8 أنواع)

## لماذا Naia؟

أدوات الذكاء الاصطناعي الأخرى مجرد "أدوات". Naia هو **"ذكاؤك الاصطناعي الخاص"**.

| | أدوات الذكاء الاصطناعي الأخرى | Naia |
|---|----------------|------|
| **الفلسفة** | استخدام الذكاء الاصطناعي كأداة | امنح الذكاء الاصطناعي نظام التشغيل. عيشوا معاً. |
| **الجمهور** | المطورون فقط | كل من يريد ذكاءً اصطناعياً خاصاً |
| **اختيار الذكاء الاصطناعي** | المنصة تقرر | 7 سحابي + ذكاء اصطناعي محلي — أنت تقرر |
| **البيانات** | محبوسة في السحابة | الذاكرة والشخصية والإعدادات كلها محلية |
| **الصورة الرمزية** | لا يوجد | شخصية VRM ثلاثية الأبعاد + عواطف + مزامنة شفاه |
| **الصوت** | نص فقط أو TTS أساسي | 5 TTS + STT + صوت ذكائك الاصطناعي الخاص |
| **النشر** | npm / brew / pip | تطبيق سطح مكتب أو USB نظام تشغيل قابل للإقلاع |

| **التكلفة** | مفاتيح API منفصلة مطلوبة | رصيد مجاني للبدء، الذكاء الاصطناعي المحلي مجاني بالكامل |

## العلاقة مع OpenClaw

Naia مبني فوق نظام [OpenClaw](https://github.com/openclaw-ai/openclaw) البيئي، لكنه منتج مختلف جوهرياً.

| | OpenClaw | Naia |
|---|---------|---------|
| **الشكل** | خفي CLI + طرفية | تطبيق سطح مكتب + صورة رمزية ثلاثية الأبعاد |
| **الجمهور** | المطورون | الجميع |
| **واجهة المستخدم** | لا يوجد (طرفية) | تطبيق أصلي Tauri 2 (React + Three.js) |
| **الصورة الرمزية** | لا يوجد | شخصية VRM ثلاثية الأبعاد (عواطف، مزامنة شفاه، نظرات) |
| **LLM** | مزود واحد | متعدد المزودين 7 + تبديل في الوقت الحقيقي |
| **الصوت** | TTS 3 (Edge، OpenAI، ElevenLabs) | TTS 5 (+Google، Nextain) + STT + مزامنة شفاه الصورة الرمزية |
| **العواطف** | لا يوجد | 6 عواطف معينة لتعبيرات الوجه |
| **الإعداد** | CUI | GUI + اختيار صورة رمزية VRM |
| **تتبع التكاليف** | لا يوجد | لوحة رصيد في الوقت الحقيقي |
| **التوزيع** | npm install | Flatpak / AppImage / DEB / RPM + صورة نظام تشغيل |
| **متعدد اللغات** | CLI إنجليزي | GUI بـ 14 لغة |
| **القنوات** | بوت خادم (متعدد القنوات) | بوت Discord DM مخصص لـ Naia |

**ما أخذناه من OpenClaw:** بنية الخفي، محرك تنفيذ الأدوات، نظام القنوات، نظام المهارات البيئي (متوافق مع أكثر من 5,700 مهارة Clawhub)

**ما بناه Naia جديداً:** Tauri Shell، نظام صور VRM الرمزية، وكيل متعدد LLM، محرك العواطف، تكامل TTS/STT، معالج الإعداد، تتبع التكاليف، تكامل حساب Nextain، نظام الذاكرة (STM/LTM)، طبقات الأمان

## البدء السريع

### المتطلبات المسبقة

- Linux (Bazzite، Ubuntu، Fedora، إلخ)
- Node.js 22+، pnpm 9+
- Rust stable (لبناء Tauri)
- حزم النظام: `webkit2gtk4.1-devel libappindicator-gtk3-devel librsvg2-devel` (Fedora)

### تشغيل التطوير

```bash
# تثبيت التبعيات
cd shell && pnpm install
cd ../agent && pnpm install

# تشغيل تطبيق Tauri (Gateway + Agent تشغيل تلقائي)
cd ../shell && pnpm run tauri dev
```

عند تشغيل التطبيق، تلقائياً:
1. فحص صحة OpenClaw Gateway → إعادة الاستخدام إذا كان يعمل، وإلا تشغيل تلقائي
2. تشغيل Agent Core (Node.js، اتصال stdio)
3. عند إغلاق التطبيق، يتم إنهاء Gateway المُشغَّل تلقائياً فقط

### الاختبارات

```bash
cd shell && pnpm test                # اختبارات وحدة Shell
cd agent && pnpm test                # اختبارات وحدة Agent
cd agent && pnpm exec tsc --noEmit   # فحص الأنواع
cargo test --manifest-path shell/src-tauri/Cargo.toml  # اختبارات Rust

# E2E (يتطلب Gateway + مفتاح API)
cd shell && pnpm run test:e2e:tauri
```

### بناء Flatpak

```bash
flatpak install --user flathub org.freedesktop.Platform//24.08 org.freedesktop.Sdk//24.08
flatpak-builder --user --install --force-clean build-dir flatpak/io.nextain.naia.yml
flatpak run io.nextain.naia
```

## نموذج الأمان

يطبق Naia نموذج أمان **الدفاع في العمق (Defense in Depth)**:

| الطبقة | الحماية |
|--------|---------|
| نظام التشغيل | Bazzite rootfs غير قابل للتغيير + SELinux |
| البوابة | مصادقة جهاز OpenClaw + نطاقات الرموز |
| الوكيل | أذونات من 4 مستويات (T0~T3) + حظر لكل أداة |
| الغلاف | نافذة موافقة المستخدم + مفتاح تشغيل/إيقاف الأدوات |
| التدقيق | سجل تدقيق SQLite (تسجيل جميع عمليات تنفيذ الأدوات) |

## نظام الذاكرة

- **ذاكرة قصيرة المدى (STM):** محادثة الجلسة الحالية (Zustand + SQLite)
- **ذاكرة طويلة المدى (LTM):** ملخصات الجلسات (مولدة بواسطة LLM) + استخراج تلقائي لحقائق/تفضيلات المستخدم
- **مهارة المذكرات:** حفظ/استرجاع مذكرات صريح عبر `skill_memo`

## عملية التطوير

```
PLAN → CHECK → BUILD (TDD) → VERIFY → CLEAN → COMMIT
```

- **BUILD = TDD** — الاختبارات أولاً (RED) -> تنفيذ أدنى (GREEN) -> إعادة هيكلة
- **VERIFY** — التأكيد بتشغيل التطبيق فعلياً (فحص الأنواع وحده غير كافٍ)
- **الالتزامات** — إنجليزي، `<type>(<scope>): <description>`
- **المنسق** — Biome (مسافة، علامات اقتباس مزدوجة، فاصلة منقوطة)
